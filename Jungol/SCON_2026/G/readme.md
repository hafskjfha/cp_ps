- **플랫폼**: Jungol
- **난이도**: 3.2/10
- **풀이 유형**: 매개변수탐색 / 이분탐색 / 그리디 / 우선순위큐

---

## 1. 문제 요약
- 문제 핵심: 
  - 서버에 순서대로 $A_i$안정도를 가지는 요청이 들어온다. 서버에는 정수 $T$의 신뢰도를 가지고 있으며 $A_i\ge T$라면 정상요청으로 간주, $A_i < T$라면 위험요청으로 간주한다. 만약 정상요청이라면 credit을 1받고, 위험요청이라면 credit이 1차감된다. 초기 credit은 0이다.
  - 처리 도중 어느 순간이라도 credit이 음수가 되면 서버는 버티지 못하고 즉시 다운된다.
  - 운영자는 예산 $B$를 가지고 있다. 위험요청에 대해서 패치를 적용하면 해당요청은 정상요청으로 바뀌지만 예산은 $T-A_i$만큼 줄어든다. 예산은 음수가 될 수 없다.
  - 서버를 끝까지 한 번도 다운시키지 않도록 할 수 있는 최대의 정수 신뢰도 $T$를 구하여라.
- 입력/출력 조건: 
- 제한 조건: $1 \le N \le 2\times 10^5, 0\le B \le 10^9, 1\le A_i \le 10^9$

---

## 2. 접근 아이디어
- 자력 풀이 시도:
  - 조건을 만족시키는 최대/최소 정수를 구하는 것이라서 이분탐색을 생각했다. 만약 신뢰도 $K$로 서버를 다운시키지 않게 운영할 수 있다면 $K-1$도 서버가 다운되지 않기 때문이다.
  - 판정함수를 생각해보자면 안정도가 $K$이상인 요청들은 그냥 처리해서 크레딧을 얻으면 되는데 $K$미만인 요청들에 대해서 효율적으로 예산을 사용해야 같은 신뢰도 $K$에 대해서 달라질 것이라고 생각했다.
  - 일단 위험요청을 처리하되 크레딧이 0일때 어떤 위험요청을 정상요청으로 패치했다고 가정하면 어떨까라고 생각했다.
  - 안정도가 높은 요청일수록 비용이 적음을 알수 있어서 안정도가 높은 요청이면 안정요청으로 바꾸는 게 좋아보였다.
  - 그래서 위험요청은 일단 패치하지 않고 -1 credit으로 넘긴다고 생각한다. 그런데 어떤 부분에서 credit이 음수가 되려는 순간, 지금까지 등장한 위험요청 중 하나를 반드시 패치해야 한다. 이때 패치 비용은 $T - A_i$이므로, 이전 요청중 $A_i$가 가장 큰 위험요청을 패치하는 것이 항상 최적이다.
  - $T$의 범위를 처음에는 $A$의 최댓값인 $10^9$으로 하였으니 WA를 받아서 적당히 범위가 틀렸음을 알게되었고 적당히 큰 수인 $10^{18}$으로 했더니 맞았다.
- 막혔던 포인트:
  - (왜 못 풀었는지, 착각한 부분 등)
- 답지 참고 후:
  - 예산 $B$가 넉넉하다면, 모든 $A_i$가 최대치($10^9$)일지라도 예산을 쏟아부어서 신뢰도 $T$를 $10^9$보다 훨씬 더 높게 만들 수 있기 때문에 이분탐색의 상한은 $10^{10} (\max(A)+B)$으로 처리해주면 된다고 한다.

## 3. 코드 정리
[정답 코드](./answer.py)
```py
import sys, heapq

def main():
    input=sys.stdin.readline
    n,b=map(int,input().split())
    a=[*map(int,input().split())]
    
    left,right=0,10**10
    ans=-1
    
    while left<=right:
        mid=(left+right)//2
        
        pq=[]
        credit=0
        cost=0
        
        for x in a:
            if x>=mid:
                credit+=1
            else:
                if credit==0:
                    if not pq or x>-pq[0]:
                        cost+=mid-x
                        credit+=1
                    else:
                        k=-heapq.heappop(pq)
                        cost+=mid-k
                        credit+=1
                        heapq.heappush(pq,-x)
                        
                else:
                    heapq.heappush(pq,-x)
                    credit-=1
        
        if cost<=b and 0<=credit:
            ans=mid
            left=mid+1
        else:
            right=mid-1
    
    print(ans)
                    
    
    
main()
```

## 4. 시간/공간 복잡도

* 시간: $O(N\log N \log (10^{10}))$
* 공간: $O(N)$
