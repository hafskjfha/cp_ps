# 문제 제목: [Count Subgrid Sum = K](https://atcoder.jp/contests/abc461/tasks/abc461_d)
- **플랫폼**: 앳코더
- **난이도**: 5/10
- **풀이 유형**: 누적합 / 투포인터

---

## 1. 문제 요약
- 문제 핵심: 
  - $H \times W$의 그리드가 있고 각칸에는 0 또는 1이 적혀있다.
  - 이때 서브그리드에 적혀있는 숫자의 합이 $K$가 되는 서브그리드의 개수를 구하세요.
- 입력/출력 조건: 
- 제한 조건: $1\le H,W \le 500, 0 \le K \le HW$

---

## 2. 접근 아이디어
- 자력 풀이 시도:
  - 2D 누적합을 먼저 떠올렸고 이를 이용하는 것이라고 생각했다.
- 막혔던 포인트:
  - 브루트포스로 풀면 $O((HW)^2)$의 시간복잡도를 가지기 때문에 최적화해야하는데 아이디어가 부족했다.
- 답지 참고 후:
  - 행(또는 열)을 2개 고정을 한다면 열(또는 행)만 움직이는 것으로 하여 2차원을 1차원으로 압축할 수 있다
  - 1차원으로 압축하면 결국 1차원 배열에서 합이 $K$인 부분 배열찾기 문제로 바꿀수 있다. 배열의 원소가 0이상이므로 누적합 배열로 바꾸면 누적합 배열은 단조증가 한다.
    - 이것은 투포인터, 이분탐색, 해시맵 3가지 방법이 있다.
    - 투포인터 풀이는 왼쪽 끝을 고정하면서 처음으로 $a[i]-a[l]\ge K$를 만족하는 $i$와 $a[j]-a[l]> K$를 만족하는 $j$를 찾으면 구간$[i,j)$에 있는 모든 인덱스 $r$에 대해 $a[r]-a[l]=K$가 성립하게 되므로 구간합이 $K$인 부분배열의 개수는 $j-i$가 된다.
    - 해시맵 풀이는 현재 부분합이 $S$일때 이전에 $S-K$가 몇번 나왔는지 안다면 누적합의 특성에 따라 그것의 개수가 구간합이 $K$인 부분 배열의 개수가 된다.

---

## 3. 코드 정리
투포인터 풀이와 해시셋 풀이를 작성하였다.\
[정답 코드](./answer.py)
```py
def main():
    input=open(0).readline
    h,w,k=map(int,input().split())
    g=[[*map(int,input().strip())]for _ in range(h)]
    ps=[[0]*(w+1)for _ in range(h+1)]
    for i in range(1,h+1):
        for j in range(1,w+1):
            ps[i][j]=g[i-1][j-1]+ps[i-1][j]+ps[i][j-1]-ps[i-1][j-1]
    
    ans=0
    
    for x1 in range(1,h+1):
        for x2 in range(x1,h+1):
            a=[ps[x2][j]-ps[x1-1][j] for j in range(w+1)]

            i,j=0,0
            for l in range(w+1):
                i=max(i,l+1)
                j=max(j,l+1)
                while i<=w and a[i]-a[l]<k:i+=1
                while j<=w and a[j]-a[l]<=k:j+=1
                ans+=j-i
            
            # c={0:1}
            # for i in range(1,w+1):
            #     s=a[i]
            #     ans+=c.get(s-k,0)
            #     c[s]=c.get(s,0)+1
                

    
    print(ans)
    
main()
```

---

## 4. 시간/공간 복잡도

* 시간: $O(H^2W)$
* 공간: $O(HW)$

---

## 5. 배운 점 / 실수

* 2차원을 1차원으로 압축시키는 아이디어를 알게되었다.
* 1차원 배열에서 누적합이 $K$인 부분 배열의 개수를 찾는 방법을 알게되었다.
* 단조증가하는 누적합 배열에서는 lower_bound / upper_bound를 찾는 문제를 투포인터로 $O(N)$에 처리할 수 있다는 점을 배웠다.
