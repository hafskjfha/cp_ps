## 1. 문제 요약
- 문제 핵심: 
  - `0`과 `1`로 이루어진 길이 $N$인 문자열 $S$가 주어진다. 
  - 인접한 두 문자를 서로 바꾸는 연산을 반복하여, 인접하면서 서로 다른 문자 쌍의 개수를 짝수로 만드려고 한다. 
  - 필요한 최소 연산 횟수를 구하라.
- 입력/출력 조건: 만드는게 불가능 하면 -1을 출력한다.
- 제한 조건: $2\le N \le 2\times10^5$

---

## 2. 접근 아이디어
- 자력 풀이 시도:
  - `00`, `11` 이면 인접하면서 서로 다른 문자 쌍의 개수가 짝수가 `01`,`10`이면 홀수니까 문자열을 분해하여 푸는 것이라고 생각하였다.
- 막혔던 포인트:
  - 문자열을 분해하여도 그것이 스왑 횟수에 영향을 미치는지 알 수 없었다.
- 답지 참고 후:
  - 인접한 서로 다른 문자 쌍의 개수는 문자열에서 값이 바뀌는 경계(0->1 또는 1->0)의 개수와 같다. 문자열을 왼쪽부터 따라가면 경계를 지날 때마다 현재 문자가 토글된다. 따라서 양 끝 문자가 같으면 토글 횟수가 짝수, 다르면 홀수가 된다.
  - 양끝값이 다르다면 왼쪽 끝 문자와 같은 문자를 오른쪽 끝까지 이동시키는 비용과 오른쪽 끝 문자와 같은 문자를 왼쪽 끝까지 이동시키는 비용중 최솟값이 답이다.
  - 다만 `01`,`10`은 어떤 문자열을 끌어오더라도 양끝값이 같게되지 못하므로 `-1`이 답이다.

## 3. 코드 정리
[정답 코드](./answer.py)
```py
def main():
    input=open(0).readline
    n=int(input())
    s=input().strip()
    if s[0]==s[-1]:return print(0)
    elif s in ("01","10"):return print(-1)
    else:
        t=n
        if s[0]=="0":
            for i in range(n):
                if s[i]=="1":
                    t=min(t,i)
                if s[n-i-1]=="0":
                    t=min(t,i)
                
        else:
            for i in range(n):
                if s[i]=="0":
                    t=min(t,i)
                if s[n-i-1]=="1":
                    t=min(t,i)
        print(t)

main()
```

## 4. 시간/공간 복잡도

* 시간: $O(N)$
* 공간: $O(N)$

## 5. 배운 점 / 실수

* 이진 문자열에서 인접한 서로 다른 문자 쌍의 개수는 문자열의 양 끝 문자의 같고 다름만으로 홀짝을 판단할 수 있다는 점을 배웠다.
* 홀짝이나 이진과 같은 것에서는 불변량 찾기가 가능한지 살펴보아야 함을 배웠다.
