## 1. 문제 요약
- 문제 핵심: 
  1. 처음에 $N$개의 레몬이 일렬로 놓여 있고 $i$번째 레몬에는 $A_i$가 쓰여있다. $A_i$는 1또는 2다.
  2. 연속된 한 개 이상의 레몬 중, 레몬에 적힌 수의 합이 3이 되는 구간을 하나 골라 구간에 포함된 모든 레몬을 모두 제거한다.
  3. 더 이상 제거할 수 있는 레몬이 없을 때까지 2번 과정을 반복한다.
  - 주어진 모든 레몬을 제거할 수 있는지 판별하라.
- 입력/출력 조건: 주어진 모든 레몬을 제거할 수 있다면 `Yes` 그렇지 않다면 `No`를 출력하세요.
- 제한 조건: $1\le N \le 2\times 10^5$

---

## 2. 접근 아이디어
- 자력 풀이 시도:
  - 합을 3으로 만드는 케이스는 2 1 / 1 2 / 1 1 1 이다.
  - 최대한 2를 먼저 사용해서 2를 다 삭제시킨다면 남은 1의 개수가 3의 배수인 것만 셀수 있게 바뀔 것 같았다.
  - 백준 9935 문자열 폭발을 사용한다면 시뮬레이션으로 풀릴 것 같았다. 다만 의도가 아니였음을 알고 다른 풀이를 모색했다.
- 막혔던 포인트:
  - 어떤 2가 삭제되지 못하는 경우가 뭐가 있는지 알지 못해 구현에 어려움을 겪었다.
- 답지 참고 후:
  - 1의 개수가 2의 개수보다 적으면 2가 삭제되지 못한다. -> `No`
  - 1의 개수가 2의 개수보다 같거나 크면 2를 무조건 다 지울 수 있다.
    - 왜냐하면 1->2또는 2->1와 같이 바뀌는 부분이 무조건 존재하고 그렇다면 이 부분에서 연쇄적으로 2를 제거해나갈 수 있기 때문이다.
    - 또한 1과 2를 짝지어 삭제하여도 1의 개수와 2의 개수의 대소관계는 유지되므로 귀납법을 이용하여 증명할 수 있다.
  - 따라서 1의개수가 2의 개수보다 많다면 (1의 개수 - 2의 개수)이 3의 배수라면 남은 1도 다 지울 수 있으므로 `Yes` 그렇지 않다면 1이 남으므로 `No`이다.

## 3. 코드 정리
[정답 코드](./answer.py)
```py
def main():
    input=open(0).readline
    n=int(input())
    a=[*map(int,input().split())]
    c1=a.count(1)
    c2=a.count(2)
    if c2>c1:print("No")
    else:print("YNeos"[(c1-c2)%3!=0::2])
    
main()
```
