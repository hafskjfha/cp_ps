# 문제 제목: [뚜따와 알람시계](https://dojoi.xyz/ko/problems/164)
- **플랫폼**: DOJ
- **난이도**: 8.5/10
- **풀이 유형**: 비트 / 그리디

---

## 1. 문제 요약
- 문제 핵심:
  - 수열 $A$에서 원하는 원소들을 뽑아 bitwise OR한 값이 $M$을 넘게 하지 않으면서 만들수 있는 최댓값을 구하시오.
- 입력/출력 조건:
- 제한 조건: $1 \le N \le 5 \times 10^5, 0\le M\le 10^{18}, 0\le A_i \le 10^{18}, A_1,A_2,\cdots ,A_N$ 은 서로다르다.

---

## 2. 접근 아이디어
- 자력 풀이 시도:
  - $M$보다큰 $A_i$는 OR취하면 $M$보다 커지니까 제외시켜야 한다고 생각했다.
  - $M$에 대해서 특정비트를 0으로 바꿔서 생각하는 건가 했지만 시간복잡도를 줄일수 없어 실패했다.
- 막혔던 포인트:
  - bitwise OR의 특성에 대해서 이해가 부족해 원소들을 필터링하는 방법과 시간복잡도를 줄이는 방법을 몰랐다.
- 답지 참고 후:
  - $M$의 i번째 비트가 1이고 OR한 값의 $i$번째 비트가 0이면 OR한 값$\le M$을 만족한다.
  - $i$번째 비트가 최초로 다르다면(M의 i번째 비트가 1이고 OR한 값의 i번째 비트가 0인 경우) 그 뒷비트$(2^{i-1},2^{i-2},\cdots,2^0)$들은 상관이 없다.
  - 그대신 앞비트$(2^i,2^{i+1},\cdots)$들은 $M$의 서브마스크(부분집합)이어야 한다.
  - 따라서 $M$이 1인 비트 위치에 대해 두 조건을 만족하는 원소들을 필터링한후 모두 OR한값들중 최댓값을 찾으면 된다.

---

## 3. 코드 정리
[정답 코드](./answer.py)
```py
def main():
    input=open(0).readline
    n,m=map(int,input().split())
    a=[x for x in map(int,input().split())if x<=m]
    
    if not a: return print(-1)
    
    ans=max(a)
    for i in range(m.bit_length()):
        if (m>>i)&1:
            temp=temp2=0
            for x in a:
                if ((m >> (i+1)) | (x >> (i+1))) == (m >> (i+1)) and ((x >> i) & 1) == 0:
                    temp|=x
                if x&m==x:
                    temp2|=x
            ans=max(ans,temp,temp2)
    print(ans)
main()
```
---

## 4. 시간/공간 복잡도

* 시간: $O(60N)$
* 공간: $O(60+N)$

## 5. 배운 점 / 실수

* 비트와 OR연산의 특성을 알게되었다.
* 비트를 어떻게 다르고 비트적으로 생각할때 어떻게 해야할지 알게되었다.
